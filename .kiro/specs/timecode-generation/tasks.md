# Implementation Plan

- [x] 1. データモデルと設定基盤の構築
- [x] 1.1 ジェネレーター設定モデルの作成
  - タイムコード生成に必要な設定を保持するモデルを作成する（フレームレート、開始時間、出力デバイスID、音量レベル）
  - デフォルト値として30fps、開始時間00:00:00:00、音量0.8を設定する
  - _Requirements: 1.2, 1.3, 3.3, 4.1_

- [x] 1.2 タイムコードソース種別の拡張
  - タイムコードソース種別にジェネレーターモードを追加する
  - 既存のソース設定モデルにジェネレーター設定を含める
  - プロジェクトファイルのシリアライズ・デシリアライズでジェネレーター設定が正しく永続化されることを検証する
  - _Requirements: 2.1, 4.1, 4.2, 4.3_

- [x] 2. タイムコードジェネレーターの実装
- [x] 2.1 ジェネレーターインターフェースの定義
  - タイムコード生成サービスのインターフェースを定義する（現在値取得、実行状態、開始・停止・リセット操作、フレーム生成イベント）
  - _Requirements: 1.1, 1.5_

- [x] 2.2 フレーム精度タイマーによるタイムコード生成
  - PeriodicTimerベースの非同期ループでフレームレートに同期したタイムコードをカウントアップする
  - Stopwatchによる経過時間追跡で累積ドリフトを補正し、フレーム落ちや重複を防止する
  - 開始時間からのフレーム単位インクリメントを行い、ドロップフレーム計算は既存のTimecodeValue演算に委譲する
  - 4種のフレームレート（24, 25, 29.97DF, 30fps）すべてで正確に動作することを確認する
  - _Requirements: 1.1, 1.2, 1.4_

- [x] 2.3 開始・停止・リセット制御
  - 開始操作で指定された開始時間からカウントアップを開始する
  - 停止操作でタイマーを停止しフレーム生成を終了する
  - リセット操作で現在のタイムコード値を設定された開始時間に戻す（生成中でも巻き戻し可能）
  - _Requirements: 1.3, 1.5, 1.6_

- [x] 3. LTCエンコーダーの実装
- [x] 3.1 (P) LTCフレームエンコード
  - タイムコード値をSMPTE LTCの80ビットフレーム（BCDフィールド + sync word）に変換する
  - 既存LTCデコーダーと同一のビット配置仕様を使用する（bits 0-3: frame units, 8-9: frame tens等）
  - ドロップフレームフラグ（bit 10）を29.97fps時に正しく設定する
  - _Requirements: 3.1_

- [x] 3.2 (P) BMCオーディオサンプル生成
  - 80ビットLTCフレームをBiphase Mark Codingで変調し、PCMオーディオサンプルを生成する
  - ビット'1'はセル内2回遷移、ビット'0'はセル境界1回遷移のルールでサンプル波形を構築する
  - 音量レベル設定に応じた振幅スケーリングを適用する
  - NAudioの音声出力パイプラインと統合するため、IWaveProviderインターフェースを実装する
  - フレームキューからリアルタイムでサンプルを供給し、キューが空の場合はサイレンスを出力する
  - _Requirements: 3.1, 3.3_

- [x] 4. TimecodeEngineへのジェネレーターモード統合
- [x] 4.1 ジェネレーターモードの起動・停止
  - 既存のタイムコードエンジンインターフェースにジェネレーター開始・リセットメソッドを追加する
  - ジェネレーター開始時に生成されたフレームを既存のフレーム処理パイプライン（Channel経由）に注入する
  - これにより既存のキュートリガーとOSCリレーがジェネレーターからのタイムコードに自動連携する
  - 停止時にジェネレーターとLTCオーディオ出力の両方をクリーンアップする
  - _Requirements: 1.5, 1.8, 2.2_

- [x] 4.2 LTCオーディオ出力の制御
  - ジェネレーター開始時にWASAPIオーディオ出力デバイスを初期化し、LTCエンコーダーからのサンプルを再生する
  - 出力デバイスの列挙（Render方向）をサポートし、ユーザーが選択したデバイスで出力する
  - ジェネレーター停止時にオーディオ出力を停止しデバイスリソースを解放する
  - 出力デバイスが利用不可の場合、LTC出力なしでジェネレーターのみ動作させる（graceful degradation）
  - _Requirements: 3.2, 3.4, 3.5_

- [x] 4.3 ソース切り替え制御
  - LTC受信モードからジェネレーターモードへの切り替え時に、現在のソースを停止してからジェネレーターを開始する
  - ジェネレーターモードからLTC受信モードへの切り替え時に、ジェネレーターとLTC出力を停止してからLTC受信を開始する
  - アクティブソースの状態を正しく追跡し、StatusChangedイベントで通知する
  - _Requirements: 2.2_

- [x] 5. ViewModelとUIの実装
- [x] 5.1 ジェネレーター制御のViewModel拡張
  - タイムコードViewModelにソース選択、ジェネレーター状態、開始時間、フレームレート、出力デバイス、音量のバインディングプロパティを追加する
  - ジェネレーターの開始・停止・リセットのコマンドを追加する
  - 出力オーディオデバイスの列挙と選択機能を追加する
  - _Requirements: 1.7, 2.1, 2.5, 3.2, 3.6_

- [x] 5.2 モード依存UIの有効・無効制御
  - ジェネレーターモード選択時にオーディオ入力デバイス選択UIを無効化する
  - LTC受信モード選択時にジェネレーター制御UI（開始・停止・リセット、開始時間、フレームレート）を無効化する
  - ソース切り替え時にUI状態を即座に反映する
  - _Requirements: 2.3, 2.4_

- [x] 5.3 ジェネレーター制御UIの構築
  - タイムコード表示ビューにソース選択コントロール（LTC受信/内部生成の切り替え）を追加する
  - ジェネレーターの開始・停止・リセットボタン、開始時間入力、フレームレート選択、出力デバイス選択、音量スライダーを配置する
  - LTC出力中の状態インジケーターを表示する
  - 既存のダークテーマに準拠したスタイリングを適用する
  - _Requirements: 1.7, 2.1, 2.5, 3.2, 3.6_

- [x] 6. DI登録と結合テスト
- [x] 6.1 サービス登録とアプリケーション結合
  - ジェネレーターとLTCエンコーダーのサービスをDIコンテナに登録する
  - タイムコードエンジンの初期化にジェネレーター依存を注入する
  - アプリケーション起動時にすべてのサービスが正しく解決されることを確認する
  - _Requirements: 1.8_

- [x] 6.2 ユニットテスト
  - ジェネレーターの各フレームレートでのカウントアップ精度を検証する（1秒分のフレーム数）
  - リセット後に開始時間へ正しく復帰することを検証する
  - ドロップフレーム（29.97fps）の正確な生成を検証する
  - LTCエンコーダーの80ビットフレーム構築を既知値と比較検証する
  - 生成設定のシリアライズ・デシリアライズ往復を検証する
  - _Requirements: 1.1, 1.2, 1.4, 1.6, 3.1, 4.1, 4.2_

- [x] 6.3 統合テスト
  - LTCエンコーダー→LTCデコーダーのラウンドトリップで、エンコードしたサンプルをデコードして元のタイムコード値と一致することを検証する
  - ジェネレーター開始時にTimecodeUpdatedイベントが正しく発火することを検証する
  - LTC受信→ジェネレーター切り替え時のイベント連続性を検証する
  - ジェネレーター設定を含むプロジェクトファイルの保存・読み込み往復を検証する
  - _Requirements: 1.8, 2.2, 3.1, 4.1, 4.2, 4.3_
